# BYCIGAR 全栈测试计划

## 一、测试目标

对 BYCIGAR 电商平台的全部前后端功能进行全面、系统的测试，覆盖：

- 所有 47 个后端 API 端点的功能正确性、边界情况和错误处理
- 所有 5 个 Pinia Store 的状态管理和异步操作
- 所有 10 个前端组件的渲染和交互
- 所有 17 个路由页面（11 公开 + 6 管理后台）的核心逻辑
- 6 条核心用户流程的端到端验证
- 安全性测试（认证、授权、注入防护）

## 二、测试工具选型

| 层级 | 工具 | 用途 | 安装位置 |
|------|------|------|----------|
| 后端 API 测试 | Go `testing` + `net/http/httptest` + `testify` | 测试所有 47 个 API 端点 | `server-go/` |
| 前端组件测试 | Vitest + Vue Test Utils + happy-dom | 测试 Stores、组件、视图 | `bycigar-vue/` |
| E2E 流程测试 | Playwright | 核心用户流程端到端验证 | `bycigar-vue/` |

### 工具配置说明

**后端（Go）**
- `testify/assert` — 断言库，简化测试代码
- `testify/suite` — 测试套件，管理共享的测试上下文
- `net/http/httptest` — 模拟 HTTP 请求，无需启动真实服务器
- 测试文件放在 `server-go/internal/handlers/*_test.go`

**前端（Vue）**
- `vitest` — Vite 原生测试框架，零配置兼容
- `@vue/test-utils` — Vue 组件挂载和交互模拟
- `happy-dom` — 轻量 DOM 环境，替代 jsdom
- `msw` (Mock Service Worker) — 拦截 API 请求，模拟后端响应
- 测试文件放在对应源文件旁：`*.spec.js`

**E2E（Playwright）**
- 支持多浏览器测试（Chromium/Firefox/WebKit）
- 自动等待、截图、录屏
- 测试文件放在 `bycigar-vue/e2e/`

| 购物车 | 2 个商品（不同分类） |
| 收藏 | 3 个商品 |
| 订单 | 1 个已完成订单（2 件商品） |

### 7.2 测试数据库管理

- 每次测试运行前清空并重新 seed
- 使用独立的测试数据库 `bycigar_test`（非生产库）
- MinIO 使用测试 bucket
- 测试结束后可选保留数据用于调试
